Programming

I'm a self-learning programmer, looking for junior jobs, mostly in frontend.
In 2014, I've decided to start using macros (VBA) to better serve my colleagues' needs by simplifying and speeding up (HR and accounting related) administrative tasks significantly. I've reached a mid-level in about 2 years, reading documentations and experimenting on over-complicated, time-consuming and monotonic work processes.
It was some years ago, when I started to learn programming more intensively and however in a wider range, too.
First, I was about to gain general knowledge with Python - and PyGame, for the fun part. Mostly using Tkinter/ttk GUIs for my desktop apps ever since then, but also tried out Kivy/kv in Java. For databases, I use SQL, MySql (or JSON, on web), SQLite - and a bit of PHP, for contact, guestbooks and such. My Python is on a stable intermediate level. As for web development, my favourite is React, but I got to know Flask, too.
After several template projects for my CV/portfolio, my Javascript (Node.js) skills have improved a lot. Besides vanilla Javascript (simple 2D canvas animations), recently I started to turn towards this specific kind of web animations (3D scene) with ThreeJS.
I use (in order of preference) VS Code, PyCharm, Sublime Text, Notepad++, IntelliJ IDEA (just Java). Git or rather GitHub for storing, handling, updating my code and stuff.
